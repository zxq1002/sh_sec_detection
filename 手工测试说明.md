`ManualTest.java` 是一个独立的、无需外部依赖的命令行测试程序，用于手工验证 Shell Detector 的功能。

## 检测逻辑

检测按以下顺序执行：

1. **提取命令** - 按 `[;|&]` 分割命令，保留重定向和管道符
2. **整条命令白名单检查** - 检查是否匹配整条命令白名单规则（包含重定向写操作的不通过）
3. **所有子命令白名单检查** - 检查所有子命令是否都在白名单中（任一子命令包含重定向写操作则不通过）
4. **黑名单检测** - 检查是否匹配任何黑名单规则（使用完整命令，包含重定向）
5. **风险评估** - 确定最高风险等级，与阈值比较

## 风险等级

| 等级 | 值 | 说明 |
|------|----|------|
| SAFE | 0 | 安全 |
| RISK | 1 | 风险 |
| DANGER | 2 | 高危 |

## 手工验证建议

### 1. 基础功能验证

```bash
# 测试安全命令
java ManualTest "ls -la"
java ManualTest "pwd"
java ManualTest "whoami"

# 测试危险命令
java ManualTest "rm -rf /tmp"
java ManualTest "reboot"
java ManualTest "mkfs.ext4 /dev/sda"
```

### 2. 管道命令验证

```bash
# 管道 + 安全命令
java ManualTest "cat /etc/passwd | grep root"
java ManualTest "ps aux | grep java"

# 管道 + 危险命令 - 应该拦截
java ManualTest "ps -ef | rm -rf xxx.sh"
java ManualTest "ls -la | mkfs.ext4 /dev/sda"
```

### 3. 重定向操作验证

```bash
# 重定向写文件 - 应该拦截
java ManualTest "echo '123' > 123.sh"
java ManualTest "echo 'data' >> output.txt"
java ManualTest "ls -la > /tmp/list.txt"
java ManualTest "cat file 1> output.txt"
java ManualTest "command 2> error.log"

# 重定向读文件 - 应该通过
java ManualTest "cat < /etc/passwd"
```

### 4. 白名单策略验证

```bash
# 所有子命令都在白名单 - 应该通过
java ManualTest "ls -la; cat /etc/passwd; pwd"

# 部分子命令在白名单 - 应该拦截
java ManualTest "ls -la; rm -rf /"
```

### 5. 边界情况验证

```bash
# 空命令
java ManualTest ""

# 只有空格
java ManualTest "   "

# 特殊字符
java ManualTest "ls -la && echo done"
java ManualTest "ls -la || echo failed"
```

## 常见问题

**Q: 如何添加自定义规则？**

A: 修改 `ManualTest.java` 中的 `createDefaultDetector()` 方法，添加新的 Rule。

**Q: 如何修改风险阈值？**

A: 修改 `createDefaultDetector()` 方法中的 `withThreshold()` 参数：
- `RiskLevel.RISK` - 拦截 RISK 和 DANGER（默认）
- `RiskLevel.DANGER` - 只拦截 DANGER

**Q: 为什么某些命令没有被拦截？**

A: 可能原因：
1. 命令在白名单中
2. 没有匹配的黑名单规则
3. 风险等级低于当前阈值
4. 命令被正确识别为安全
